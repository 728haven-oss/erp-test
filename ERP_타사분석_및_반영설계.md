**작성일:** 2026-07-02
**조사 대상:** 국내 — 이카운트(ECOUNT), 더존 WEHAGO/Smart A, 아이퀘스트 얼마에요 · 일본 — 테크노아 TECHS-S NOA, 다이코 rBOM, OBC 봉행(奉行) 시리즈, freee, 머니포워드 클라우드 · 관행 — 분할결제 세법(중간지급조건부), 수정세금계산서 6종, 통장대사 도구(볼타·캐시노트 등)

## 1. 문서 전환(상호작용) — 모든 ERP의 공통 골격

| 제품 | 전환 방식 | 특징 |
|---|---|---|
| 이카운트 | 판매입력 화면 상단 [주문] 버튼 → 미판매 주문 팝업에서 선택해 불러오기 | 견적대비 미주문·주문대비 미판매 **잔량 자동 계산**, "미판매현황" 보고서로 추적 |
| 더존 Smart A | [견적적용]/[출고적용] 버튼 → 다이얼로그에서 **건별(문서) 탭 / List(품목) 탭** 선택, 체크박스로 끌어옴 | 잔량 0이면 진행상태 자동 "완료"+마감. **진행중이면 품목 삭제 불가, 완료면 수정 잠금** |
| 얼마에요 | 견적→주문→발주→입고→출고 자동 연결, 분할 입출고 지원 | 출고만 하면 거래명세표·세금계산서 자동 생성 (건별/월합계 거래처별 선택) |
| OBC 봉행 | 과거 견적 검색→원클릭 복사, 성약 견적을 수주전표 또는 매출전표로 "전표 리레이" | 수주 건너뛰고 바로 매출 가능 |
| TECHS-S | 인합(리드)→견적→수주 복사. 견적 상태: 상담중/개정(改版)/수주/보류/실주 | 견적 버전관리가 상태값으로 존재 |

**반영 결정:**
- 전환은 "버튼 + 원본 참조" 방식: 견적 [수주 전환], 수주 [납품 생성]·[발주 생성], 발주 [입고 처리], 납품/회차/수주 [계산서 발행].
- **부분 납품 = 라인별 수량 편집 + 잔량 추적** (더존식). 누적 납품수량이 수주수량에 도달하면 납품완료 자동 판정.
- 모든 문서 상세에 **"연결 문서" 패널** (원본 견적↔수주↔납품↔계산서↔회차↔입금배분↔발주) — 어느 문서에서든 전체 체인 왕복 가능. 이것이 요청하신 "메뉴 상호작용"의 구현체.
- 견적 상태에 실주(lost) 추가 (TECHS 패턴) — draft/sent/converted/lost.

## 2. 수금·미수금 — "회수예정표" 개념 채택

- 이카운트·더존·얼마에요 모두 수금은 **거래처 단위 금액 입력**이 기본, 건별 반제는 선택 기능. 우리 설계(수주별 회차에 배분)가 오히려 더 정밀 — 일본 OBC 봉행의 **회수예정표(回収予定表)** 방식과 동일 구조.
- OBC 봉행 입금소거: 회수예정을 먼저 만들고 → 은행 명세 자동취득 → 계좌·의뢰인명으로 자동 매칭 → **체크만 하면 입금전표 자동 기표**. 부분입금·선수금 충당 지원. **송금수수료 차액·단수차이 자동 계상**(청구액과 입금액이 수백 원 어긋나는 것을 허용 처리).
- 볼타(국내): 금액·날짜·거래처명·사업자번호 종합 비교 → 명확하면 자동 연결, 불확실하면 **후보 우선순위 정렬 제시 + 담당자 1클릭 확정**. 합산 입금(1입금:N건)·분할 입금 지원.
- 얼마에요: **외상매출 연령표(aging)** — 장기 미수 추적.

**반영 결정:**
- 수금계획 회차 = 회수예정표. 대사 화면은 "왼쪽 통장 명세 / 오른쪽 추천 후보" 인박스형(freee/MF 패턴).
- 자동 추천 점수: **금액 일치(최우선) > 거래처명·입금자명 유사도 > 예정일 근접** + 거래처별 **입금자명 별칭 학습**(이카운트의 거래처코드 자동 채움 패턴).
- **차액 허용 처리**: 배분 시 잔여 소액(수수료 등)을 '수수료/잡손실'로 털어내는 옵션 (OBC 패턴).
- 부분입금(회차 잔액 유지), 분할 배분(1입금→N회차), 매입 출금 대사, '기타'(급여·경비 등 매입 외) 분류 지원.
- 조회 메뉴에 거래처별 미수 + **경과월수(aging)** 표시.

## 3. 세금계산서 — 세법 요건 반영

- **분할결제 세법**: 계약금 지급일 다음날~잔금일이 6개월 이상이면 **중간지급조건부** → 공급시기는 "대가의 각 부분을 받기로 한 때" = **회차별 계산서 발행이 세법상 의무**. 작성일자는 입금일이 아니라 **계약상 지급예정일** 기준(지연발급 가산세 위험). → 우리 "회차별 발행" 구조가 세법과 정확히 일치. 수주에 중간지급조건부 판정 표시 추가.
- **수정세금계산서 6종**: 소급형(기재정정·이중발급·내국신용장: 당초 작성일자, 부(-)+정(+) 2장 또는 부 1장) vs 신규사건형(공급가액변동·계약해제·환입: 사건 발생일 작성, 차액 1장). 얼마에요·더존 공통 UX: **원본 조회 → 사유 선택 → (-)/(+) 자동 생성**.
- 이카운트 상태 UX: 발행 진행 5단계 원형 표시 + **상단 상태별 건수 클릭 → 필터링**.
- 부가세 단수: 법정 강제 없음, 반올림 권장. 라인 세액 합 = 헤더 세액 정합 필수.

**반영 결정:** 발행근거 3택1(수주전체/납품/수금회차) 유지, 수정계산서는 사유 6종 선택 → 부(-) 자동 생성 + 원본 참조 링크. 상태 배지 클릭 필터. 세액은 라인별 반올림 후 합산.

## 4. 수주별 원가·손익 — 일본식 "예산 vs 실적" 채택

- TECHS-S: 수주(제번)별 원가를 **예산·실적·최종예측 3값 그래프**로 제작 중에 표시 → 적자 수주 조기 발견. **대표수주번호**로 추가공사·무상대응까지 묶어 안건 손익 집계.
- 이카운트: 프로젝트 태깅 방식, 공통비 자동 안분 없음(수동) — 소기업 현실.
- 표준 손익 컬럼: 수주번호 | 거래처 | 계약금액 | 자재비 | 외주비 | 원가계 | 이익 | 이익률(%) + 수금액/미수잔액 병기.

**반영 결정:** 수주 손익 = 계약액 vs 연결 발주·입고 실적(직접 자재비) + 이익률, 진행 중에도 표시. 노무비·경비는 범위 외 명시. 수주 상세에 원가 요약 패널.

## 5. 목록 화면 UX 공통 패턴

- 이카운트: 검색 패널(기간·거래처·품목·**금액 범위**), 텍스트 링크 드릴다운(거래처명→원장), 체크박스 선택→하단 일괄 버튼, 상태 숫자 클릭→필터.
- 더존: 상태값(미진행/진행중/완료)에 따른 수정·삭제 잠금.
- 일본 공통: 업무 순서대로 메뉴 배열, 청구·입금이 매출과 분리된 독립 메뉴.

**반영 결정:** 전 목록에 검색+상태 필터+상태 배지, 배지/숫자 클릭 필터, 거래처명 클릭→원장 드릴다운, 대시보드 숫자 클릭→해당 목록 이동.

## 6. 우리 설계가 타사보다 나은 점 (유지)

미수금 무저장(계산값), 회차별 수금계획(국내 3사엔 없는 정밀도 — 일본 봉행급), 품명 스냅샷, 통장 원본 불변+배분만 취소/재배분, 상태 기반 수정 통제. 모두 유지.

## 7. HTML 테스트 버전 구현 명세 (요약)

7개 메뉴 전체, 단일 HTML(브라우저에서 바로 열기), localStorage 저장(새로고침 유지) + 샘플 데이터 + 초기화 버튼. 금액 정수(원), 부가세 라인별 반올림, 문서번호 자동 채번(Q2026-0001 등), 상태 통제 규칙(A-2) 실동작, 통장 CSV 업로드/붙여넣기 + 중복 감지 + 자동 추천 대사 + 분할/부분 배분 + 차액 털기, 수정계산서 6종, 연결 문서 패널, 거래처원장(이월+잔액), 수주 손익, 수금현황(aging), 대시보드 클릭 이동.
