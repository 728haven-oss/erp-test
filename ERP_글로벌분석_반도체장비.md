# 글로벌 ERP 분석 종합 — 반도체 장비 제작사 관점

**작성일:** 2026-07-02
**조사 범위:** 한국(이카운트·더존·얼마에요·영림원) / 일본(TECHS-S·rBOM·OBC봉행·freee·MF) / 북유럽(Monitor·IFS·Visma·Fortnox·e-conomic) / 미국(JobBOSS²·E2·Global Shop·ProShop·Odoo·NetSuite·Acumatica) / 중국(용우 U8+/T+·금접 성공·정첩 易飞/E10)
**목적:** 5인 이하 반도체 장비 제작사(개별 수주생산)용 자체 ERP의 방향성 결정

---

## 1. 전 세계 공통 결론 — 수주생산 ERP의 골격은 하나다

지역과 무관하게, 개별 수주생산(ETO/MTO) 제조 ERP는 전부 같은 골격으로 수렴한다:

**"수주 1건 = 원가 축 1개"** 그리고 **"문서는 재입력 없이 이전 문서에서 전환"**.

- 일본 TECHS: 製番(제조번호)관리 — 수주번호에 BOM·발주·실적·원가 전부 연결
- 스웨덴 IFS: DOP 페깅 — 수주 1건이 오더 트리 1개, 원가 집계 단위 1개
- 미국 JobBOSS/E2: Job number — 모든 트랜잭션에 잡 번호 강제 태깅
- 중국 U8/금접: 分批法(오더법) — 생산오더 자체가 원가 대상
- 한국 영림원: 프로젝트 원가 개별법 — "프로젝트 업체는 개별법"이 공식 지침

우리 설계(수주 → 발주 연결 → 자재비 집계)는 이 글로벌 표준과 정확히 일치한다. **v1의 방향은 옳았다.** v2 이후는 이 축을 강화하는 방향으로 간다.

## 2. 승인(결재) — 지역별 3가지 유형과 우리의 선택

| 유형 | 대표 | 방식 | 5인 규모 적합성 |
|---|---|---|---|
| **A. 게이트형** (한국 영림원) | K-System | "품의 승인 전 발주 생성 불가" — 승인이 다음 단계 생성의 잠금장치. 결재선 대신 **권한자+승인 버튼** | ◎ 구조 그대로 채택 |
| **B. 예외형** (북유럽 Monitor, 미국 Odoo) | Monitor EIM, Odoo | 전건 결재 금지. **금액 한도 1개 + 승인자 1명**, 한도 이하 자동 통과. 미국 실무 원칙: "경영진 승인은 전체 거래의 20% 미만이어야 정상" | ◎ 한도 로직 채택 |
| **C. 전건형** (중국) | 금접·용우 | 모든 문서에 审批. 단 "종심(최종승인) 노드 통과 = 다음 문서 생성 권한"이라는 규칙이 명확. 메신저(딩톡/위챗) 카드 승인 — **승인자는 ERP에 로그인하지 않는다** | △ 전건은 과잉, 종심·메신저 개념만 차용 |

**v2 결정:** A+B 결합 — ①발주서: 금액 한도(설정값) 초과 시에만 승인 필수, 승인 전 **입고 처리·이메일 발송 차단**(게이트). 한도 이하 자동 통과. ②견적서: 대외 제출 문서이므로 발송·수주전환 전 승인(설정으로 끄기 가능). ③결재함 화면: "내가 승인할 문서 / 내가 올린 문서 상태"(영림원 K-SMART의 최소 단위). ④승인/반려/의견 3요소만. 다단계 결재선·그룹웨어 연동은 배제(전 지역 조사가 5인 규모엔 과잉이라고 일치).

## 3. 이메일 — "문서 화면에서 원클릭 + 타임라인"이 세계 표준

- **Odoo(최고 참고 사례):** 문서 화면에 [Send by Email] → 템플릿 자동 채움 + PDF 첨부 → 발송·회신·메모가 문서 하단 **타임라인(채터)에 자동 축적**. "별도 메일함 뒤지기 제거 — 소기업 체감효과 1위 기능."
- **Monitor:** 거래처 마스터에 **문서 종류별 수신 이메일**을 저장 → 원클릭 발송. 발주확인 미회신 독촉 리스트 표준 탑재.
- **일본/중국:** 이메일은 보조 채널화 추세(알림톡·딩톡·위챗). 한국도 카카오 알림톡이 동일 위치.

**v2 결정:** 무료 구간 = ①거래처 이메일 마스터 ②견적서·발주서·거래명세서 화면 [이메일 발송] 버튼 — 제목·본문 자동 생성 → **mailto로 기본 메일 앱 호출**(서버 불요) ③발송 기록을 문서별 타임라인으로 저장. 유료/서버 구간(자리만) = SMTP·API 자동 발송, 수신 열람 추적, 알림톡. 발송 전 결재 게이트 통과 필수.

## 4. 자재 흐름 — "Buy-to-job + 공용재고 불출" 이원 구조

미국 잡샵 표준(JobBOSS/E2)이 가장 명쾌하다:

1. **Buy-to-job (프로젝트 직송 구매):** PO에 잡 번호 지정 → **입고 확인 = 잡 원가 반영** (재고를 거치지 않음). 장비 1대 수주생산에서는 구매의 대부분이 이 경로.
2. **공용재고 경유:** 소모품·공통 자재는 입고 → 재고 적립 → **사용처리(불출) 시점에 잡 원가 반영**. E2는 바코드로 사용 시점 포스팅.

여기에 영림원 반도체장비 패키지의 **"공용재고 프로젝트 대체"**(잉여 자재를 다른 수주에 투입 전환 — 장비사 원가절감 핵심)와 易飞 **LRP**(수주 1건 기준 소요 전개), Monitor **부족자재 리스트**(확보 점검 → 불출 보고 2단계)가 붙는다.

**v2 결정:** ①수주 연결 발주 = 입고 시 자동으로 그 수주 원가(buy-to-job, v1에 이미 구현) ②공통 발주 = 자재 재고 적립 ③**사용처리(불출)** 화면: 재고에서 수주(프로젝트)로 불출 → 이동평균 단가로 그 수주 원가에 가산 ④수주 화면에서 **자재비 구성(직납 vs 불출) + 투입 내역 전체 이력** 확인 = "프로젝트 전체 단가". ⑤원가 인식 시점: 입고 즉시/불출 즉시(영림원의 월마감 단가계산은 5인 규모 과잉 — 이동평균 즉시 인식으로 단순화).

## 5. 반도체 장비 제작사 특화 시사점

- **영림원 K-System Ace 반도체장비 패키지**가 가장 직접적 선례: 프로젝트별 수불, **프로젝트 간 자재 대체**, **호기별(장비 1대별) 원가**, 프로젝트 예산/실적, A/S 관리, 수주 Forecast.
- **일본 TECHS 代表受注No.:** 본계약+추가공사+무상대응을 묶어 안건 단위 손익 — 장비업의 "설치 후 추가작업" 현실 반영.
- **ProShop(미국, 반도체 고객 보유):** 잡 화면 상단에 [견적원가 | 실적원가 | 차이%] 상시 노출, 1원 지출 즉시 갱신.
- **미국 ETO 관행:** 마일스톤 청구(계약금-중도금-검수잔금)가 수주에 내장 — v1 수금계획과 동일.
- **검수(検収) 기준:** 일본·한국 공통 — 시운전·검수 후 잔금/매출 확정. 반도체 장비는 Set-up + A/T Sign 후 잔금이 전형.

**v3+ 후보로 이어지는 것:** 호기별 관리(1수주 다(多)장비), 수주별 원가 예산(견적원가) 등록 → 예산 vs 실적 vs 예측 3값(TECHS), A/S·설치 이력, 공용재고 프로젝트 대체 전용 화면, BOM(자재명세) 관리와 LRP식 소요 전개, 알림톡/메신저 승인.

## 6. 은행 대사 보강 (참고)

Visma의 신뢰 원칙 — **"100% 확실할 때만 자동, 애매하면 '제안'으로 표시 후 사용자 확인"** — v1의 추천+1클릭 확정 구조와 일치. e-conomic의 "규칙 등록 시 특정 거래 자동 기표"(예: 매월 급여 → 자동 기타처리)는 v3+ 후보.
