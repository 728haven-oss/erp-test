# 자체 ERP 격차 분석 및 기능 제안서

**작성일:** 2026-07-02 · **목적:** 타 ERP 대비 부족한 점을 조사·분석하여 다음 버전 후보를 제안 — 검토 후 적용 여부 결정용
**조사:** 국내(이카운트·더존·얼마에요·영림원·경리나라) / 일본(TECHS·rBOM·봉행·freee/MF·사이보우즈) / 미국(JobBOSS²·E2·ProShop·Global Shop·Odoo·NetSuite·Acumatica) / 북유럽(Monitor·IFS·Fortnox) / 중국(용우·금접·정첩) + 이번 격차 전용 조사 14회(출처 각 항목 명기)

## 1. 현재 보유 기능 (v3 기준 — 격차 판단의 기준선)

견적→수주(수금계획 회차)→납품→세금계산서(수정 6종·양식 인쇄), 발주→입고→자재재고→불출→사용처리, RFQ 공급사 포털(회신·DC 협상·임시/확정 명세서·긴급요청), 결재(금액한도 게이트+메일 통지), 메일함(수신함=처리 큐), 이메일 발송(기록·용도별 주소), PRJ 관리(총액·매입·공수/인력비용·이익·완성율), 호기·도면 링크, 마진/DC 미리보기(업체별·기간별 단가+최근 매입가 원가 참조), 일정 달력+데드라인, 통장 대사(추천 매칭·은행 표기), 현금/전산 장부 분리, 거래처(담당자·용도별 이메일·내역 조회), 계정(ID/PW/이메일), 본사 설정(인증서·홈택스·통장 자리), 백업(JSON+일일 스냅샷), 모바일 CSS.

## 2. 부족한 점 — 제안 목록 (우선순위순)

### 제안 1. 검사성적서 (수입·출하 검사) — 필요도 상 ★적용 강력 권장
반도체 장비 공급망에서는 납품 시 치수검사 성적서·CoC(적합증명서) 첨부가 사실상 납품 조건. ProShop은 검사계획 단위로 관리하고 입력된 치수로 성적서를 자동 생성. **제안:** 납품(거래명세서)에 "검사 기록" 섹션(항목·기준·측정값·판정) + 성적서 양식 인쇄. 입고 검수도 동일 구조 재사용(현재 입고 첨부의 확장).
출처: ProShop QMS, 1factory 반도체 장비 품질.

### 제안 2. 견적 사전원가(BOM 임포트) — 필요도 상
개별 수주생산의 손익은 견적에서 결정됨. TECHS는 부품마스터 등록 없이 CAD/엑셀 부품표를 그대로 임포트하고 과거 유사 안건 원가를 복사해 견적 산출. **제안:** 견적서에 "원가 내역(BOM) 붙여넣기"(엑셀 복사→라인 생성) + 과거 수주 원가 복사 버튼 + 사전원가 vs 견적가 마진 표시(이번에 넣은 마진 그리드의 확장). 공수 예상치 포함 시 v3의 실적 공수와 "예상 vs 실적" 비교 완성(TECHS 예산/실적/예측 3값).
출처: TECHS-S, MS Dynamics BOM 계산.

### 제안 3. 월마감 잠금 + 감사 로그 — 필요도 상
Odoo는 잠금일(Lock Date) 이전 전표의 수정을 차단, QuickBooks는 마감일+비밀번호. 대사·장부까지 갖춘 시스템에서 과거 전표가 자유 수정되면 대사 결과가 무의미해짐. **제안:** 설정에 "마감일" — 그 이전 날짜의 문서 수정·삭제 차단(관리자 예외) + 주요 변경 로그(누가·언제·무엇을). 비용 대비 효과가 가장 큰 통제.
출처: Odoo Lock Date, QuickBooks 기간 잠금, Fasoo Audit Trail.

### 제안 4. 재고실사·재고조정 — 필요도 상
이카운트는 실사전표(장부 vs 실사 수량) → 조정전표 2단계로, 임의 조정 불가·증적 보존. 자재재고·불출이 이미 있으므로 실사 경로가 없으면 장부재고가 몇 달 만에 현실과 어긋남. **제안:** 자재 탭에 [재고실사] — 실사 수량 입력 → 차이 자동 조정 전표(사유 기록).
출처: 이카운트 재고조정(중원컨설팅).

### 제안 5. A/S·설치 이력 (호기=시리얼) — 필요도 상
장비사는 납품 후 유상/무상 A/S 분쟁과 재발 고장 파악에 시리얼별 이력이 직결. 이미 호기 개념이 있어 "호기에 서비스 기록(방문일·증상·조치·교체부품·유/무상)"만 붙이면 됨. TECHS 代表受注No.(본계약+추가작업 묶음 손익)와 결합 가치.
출처: Salesforce/Dynamics Field Service, FieldPulse.

### 제안 6. 결제수단 확장 (전자어음·구매카드) — 필요도 중
더존은 받을어음을 만기일별 관리. 발행·유통은 은행이 하므로 ERP에는 수금계획 회차에 "결제수단(현금/어음/구매카드)+만기일+상태" 필드만 있으면 됨. 어음 거래가 실제 있는지에 따라 우선순위 결정.
출처: WEHAGO 받을어음현황, IBK B2B.

### 제안 7. 수출 서류 (CI/PL·환율) — 필요도 중 (해외 고객 생기면 상)
이카운트는 판매전표에서 Commercial Invoice/Packing List 즉시 출력, 통화·환율을 전표에 저장. **제안(최소):** 거래처에 통화 필드 + 견적/명세서 외화 표시 + CI·PL 양식 2종 추가(기존 양식 인쇄 체계 재사용). 수출신고는 관세사 경유가 합리적.
출처: 이카운트 수출관리, 트레드링스.

### 제안 8. KPI 경영 대시보드 — 필요도 중
데이터(견적·수주·매출·완성율·공수)가 이미 다 있어 차트 페이지 하나로 구현 가능: 월매출 추이, 견적→수주 전환율(Win Rate), 수주 파이프라인, 프로젝트 마진 분포.
출처: K-System BI, 뉴스젤리 KPI 시각화.

### 제안 9. AI 활용 (저비용 구간만) — 필요도 중
실익 순: ①통장 대사 추천의 학습 강화(이미 별칭 학습 보유 — 금액 분할 패턴 학습 추가) ②거래명세서/영수증 사진 OCR 입력(얼마에요 AI매입거래 패턴 — API 비용 소액) ③LLM으로 견적서 문구/메일 초안 생성. 자동 분개·세무신고는 세무사+상용(더존 ONE AI) 영역이라 자체 구현 비권장.
출처: 더존 ONE AI, 얼마에요 RPA, 한국딥러닝 AI OCR.

## 3. 이번에 이미 반영되어 제외된 항목

공수/노무비 원가(제안→PRJ 관리로 구현 완료), 시기·업체별 단가(특별단가+최근 매입가 참조 구현), DC/마진 가시화(견적·명세서 그리드+공급사 포털 마진), 메일함, 현금/발생 장부 분리.

## 4. 권장 적용 순서 (검토용)

| 차수 | 항목 | 이유 |
|---|---|---|
| 1차 | 마감잠금+감사로그(3) · 재고실사(4) | 기존 데이터의 신뢰성 담보 — 구현 작고 효과 큼 |
| 2차 | 검사성적서(1) · A/S 이력(5) | 반도체 장비 납품·사후 관행 직결 |
| 3차 | 견적 사전원가/BOM(2) | 공수·단가 체계가 자리 잡은 뒤 얹으면 완성도 최대 |
| 4차 | 어음(6)/수출(7)/KPI(8)/AI(9) | 거래 발생·여력에 따라 선택 |

각 항목은 승인 주시면 해당 영역 타 ERP 화면 수준 상세 조사 → 설계 → 구현 순으로 진행합니다.

## 반영 현황 (승인 후 구현 완료)

**상 5건 — 전부 구현:** ①검사성적서(수입/출하 검사 기록+성적서 인쇄, ProShop QMS) ②견적 사전원가(BOM/엑셀 붙여넣기 임포트 + 과거 프로젝트 실적원가 비례 배분 복사, TECHS) ③월마감 잠금+감사 로그(설정에서 마감일 지정 — 이전 일자 전표 차단·관리자 예외 시 로그, 주요 행위 감사 기록, Odoo Lock Date) ④재고실사·조정(장부 vs 실사 → 차이 조정 전표·증적, 이카운트 2단계) ⑤A/S·설치 이력(호기=시리얼별 유/무상 서비스 기록, Field Service).

**중 4건 — 전부 구현:** ⑥결제수단(수금 회차에 현금/전자어음(만기일)/구매카드) ⑦수출 서류(CI/PL 영문 양식 인쇄 + 거래처 통화 필드) ⑧KPI 대시보드(장부>📊: 월매출 추이 차트·견적→수주 전환율·파이프라인·마진 분포) ⑨AI(설정에 기능별 상태·비용 명기 — 대사 학습은 가동 중/무료, OCR·LLM은 유료 API 자리) + 💳카드 사용 내역 조회(CSV 업로드·카드별/월별, 자동수집은 유료 명기).

유료·가입 필요 항목 전체 명세는 개발 로드맵 문서의 "비용 경계표" 참조.
