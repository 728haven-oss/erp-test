# erp_test.html 테스트 가이드

**실행:** `erp_test.html`을 브라우저(크롬/엣지)에서 더블클릭으로 열면 됩니다. 설치 불필요. 데이터는 브라우저(localStorage)에 저장되어 새로고침해도 유지됩니다. 좌측 하단 [샘플로 초기화]로 언제든 처음 상태로 되돌릴 수 있습니다.

샘플 데이터에는 진행 중인 수주 2건(대한정밀 5,170만 / 스마트팩토리 1,540만), 발행된 계산서, 미대사 통장내역 4건이 미리 들어 있습니다.

## 권장 테스트 시나리오 (설계서의 전체 흐름 1바퀴)

1. **홈**: 총 미수금 43,890,000원 확인 → 카드 클릭 시 해당 화면으로 이동하는지.
2. **견적→수주**: 영업 > 견적서에서 Q2026-0002(테크노밸리) [수주 전환] → 견적이 읽기전용으로 잠기고 수주가 생기는지.
3. **수금계획**: 새 수주 화면에서 [30/40/30 자동 생성] 또는 회차 추가 후 비율% 입력 → 합계≠계약액이면 경고 뜨는지.
4. **납품(부분)**: 수주 SO2026-0001에서 [납품 생성] → 수량을 줄여 부분 납품 → 라인의 "잔량"이 추적되는지, 전량 납품 시 상태가 자동으로 [납품완료]가 되는지.
5. **계산서 발행**: 수금계획 회차의 [계산서 발행] → 회차가 [청구]로 바뀌고 금액이 잠기는지(🔒). 발행 근거 3택1(수주전체/납품/회차) 모두 시험.
6. **통장 대사**: 자금/통장대사 > 통장 내역 → 7/1 "(주)대한정밀" 20,680,000원 [배분] → **중도금 회차가 1순위로 자동 추천**되는지 → 확정 시 회차가 [수금완료]로.
7. **부분입금·별칭 학습**: "김재성" 3,000,000원을 스마트팩토리 잔금에 배분 → 부분입금 처리 + "김재성"이 거래처 별칭으로 학습되는지(다음부턴 자동 추천).
8. **기타 처리**: "급여 6월분" 출금을 [기타]로 → 미대사 건수가 줄어드는지.
9. **CSV 업로드**: [CSV 업로드/붙여넣기]에 `2026-07-05,테크노밸리,7920000,0` 붙여넣기 → 같은 줄 재업로드 시 중복 차단되는지.
10. **구매·원가**: 발주 PO2026-0002 [입고 처리](부분 입고 가능) → 수주 SO2026-0001 화면의 자재비·이익률이 실시간으로 변하는지 → 입고에서 [계산서 수취] → 미지급금 반영.
11. **수정계산서**: 세금계산서 목록에서 [수정계산서] → 사유 6종별로 (-)1장 또는 (-)+(+)2장이 생기는지.
12. **통제 규칙**: 발행 계산서·입금이 있는 수주를 [수주 취소] → 차단 안내(배분 취소→계산서 취소→수주 취소 순서 강제)가 뜨는지.
13. **조회/장부**: 거래처 원장(전기이월+잔액), 수주별 손익(이익률 색상), 수금계획 현황(연체 필터), 미지급 현황.

## 자동 검증 완료 항목

배포 전 브라우저 시뮬레이션으로 159개 항목을 자동 검증했습니다: 31개 화면 렌더(변경 전/후), 견적 전환, 회차 합계, 부분납품·초과납품 차단, 발행-청구 상태 전이, 자동 추천 1순위 정확도, 부분입금, 별칭 학습, CSV 중복 차단, 배분 취소 시 상태 복귀, 수정계산서 6종 분기, 취소 가드, 부분입고, 자재비 집계, 미지급 반영.

## v2·v3 추가 시나리오 (erp_test.html = 최신 v3)

14. **사용자 전환·결재**: 좌측 상단에서 "박담당"으로 전환 → 발주서 PO2026-0002가 [결재중]이라 입고 버튼이 잠긴 것 확인 → "김재성(대표)"로 전환 → 결재함에서 승인 → 입고 가능해지는지.
15. **자동승인 한도**: 100만원 이하 발주 작성 → 저장 즉시 [자동승인]. 결재함 하단에서 한도 변경.
16. **이메일**: 견적서 [메일] → 본문 자동 생성 → [메일 앱 열어 보내기] → 발송 이력이 문서에 남는지.
17. **RFQ 협업 루프**: 구매 > 견적요청(RFQ) → RFQ2026-0002 [포털(공급사 화면)] → 품번·단가·납기 기입 → [회신 제출] → [발주 전환] → 회신 단가로 발주 생성되는지.
18. **임시/확정 명세서**: 발주 상세 [공급사 포털] → 준비 수량 수정(임시 저장) → [부분 출하 확정] → 발주 화면에 ✅확정 배너 → 입고 처리 시 확정 수량 안내.
19. **긴급 입고요청**: 발주 상세 [긴급 입고요청] → 포털에 🚨 배지 표시.
20. **입고 첨부**: 구매 > 입고 목록에서 입고 건 클릭 → [첨부 추가] → 사진 선택 시 미리보기와 함께 저장.
21. **호기·진척 시각화**: 수주 상세 → 호기 카드에서 조립 완성도 % 입력, 도면번호 입력 후 [열기↗] → 우측 4개 진행바(입고율/사용률/조립/완성율)가 실시간 변동. 대시보드 제작중 목록에도 완성율 미니바.
22. **자재 사용(투입)**: 수주 상세 [사용(투입) 처리] → 가용 초과 차단, 투입 시 사용률·완성율 갱신.
23. **백업/복원**: 좌측 하단 [백업(JSON)] 파일 저장 → 데이터 수정 → [복원/스냅샷]에서 JSON 복원 또는 일일 스냅샷(7일 자동 보관) 복원.
24. **거래처 상세(내역 허브)**: 거래처 목록에서 상호 클릭 → 누적 매출/매입/미수/미지급 카드 + 이 거래처와 오간 전 문서(견적·수주·납품·발주·입고·계산서·입금) 탭 필터 조회.
25. **담당자(소속) 관리**: 거래처 상세에서 담당자 추가·수정·삭제 — 담당자 이메일이 메일 발송 수신자 후보로 등장.
26. **용도별 이메일**: 거래처 수정에서 "거래명세서용"과 "홈택스 세금계산서용" 이메일 분리 등록 → 세금계산서 [메일]은 세금용, 명세서/견적 [메일]은 문서용 주소가 자동 선택되는지.

## 이 테스트 버전의 한계 (실개발 시 Flask+DB로 옮길 것)

단일 브라우저 저장(여러 PC 공유 불가 — 실버전은 Flask 서버 1대+브라우저 접속), 로그인/사용자 식별 없음, 인쇄 양식(견적서·명세서 출력물) 미구현, 홈택스·오픈뱅킹은 자리만(내부기록), 견적서 rev 관리·첨부파일 미구현.
